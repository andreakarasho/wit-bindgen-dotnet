# WitBindgen E2E test harness

A **real** end-to-end test harness that exercises WIT features with actual round-trip
**value** assertions (not codegen string checks), over the real `wasmtime` runtime, using
generated host bindings calling a NativeAOT-LLVM guest component.

## Layout

- `wit/e2e-test.wit` â€” package `e2e:test`, world `e2e-test`, interface `test-types` (exports).
- `wit/deps/e2e-commons/commons.wit` â€” package `e2e:commons`, interface `types` (records,
  enum, flags, variants). Demonstrates cross-package `use`. It lives in a `deps/` subdirectory
  because the WitBindgen guest generator rejects two different packages in one directory.
- `Guest/` â€” wasi-wasm NativeAOT-LLVM project. References the in-repo
  `WitBindgen.SourceGenerator` (analyzer) + `WitBindgen.Runtime`, feeds the WIT via
  `WasmComponentTypeWit` / `AdditionalFiles`, and implements every export in `GuestImpl.cs`.
- `Host/` â€” xUnit (net10.0) project. References `external/wasmtime-dotnet` (`Wasmtime` +
  `Wasmtime.SourceGenerator` analyzer), feeds the same WIT, and uses the **generated**
  `Wit.E2e.Test.E2eTestExports` bindings to call each export and assert the returned value.

## Running

```bash
# 1. Build the guest component (NativeAOT-LLVM; slow, minutes):
dotnet publish tests/WitBindgen.E2E/Guest -c Release
# -> tests/WitBindgen.E2E/Guest/bin/Release/net10.0/wasi-wasm/publish/WitBindgen.E2EGuest.wasm

# 2. Run the round-trip tests (fixture auto-resolves the wasm path):
dotnet test tests/WitBindgen.E2E/Host -c Release
```

The fixture (`WasmComponentFixture`) resolves the `.wasm` via, in order: the
`WITBINDGEN_E2E_WASM` env var, a copy next to the test assembly, or the guest publish output
discovered by walking up from the test assembly. Each test gets its own component instance so
a trap in one call cannot poison ("cannot enter component instance") the others.

## Skipped features (toolchain limitations)

Each was confirmed by reading the generated code and/or running the real test, then removed
from the shared WIT so both projects build with 0 errors.

| Feature | Reason |
|---|---|
| `char` | Guest generator emits `EchoChar(int)` calling a `uint` partial without a cast (CS1503). |
| `list<u8|u32|f64>`, `sum-list-*` | Primitive-list marshaling returns garbage values at runtime (e.g. `[0,128,255]` came back as `[44,219,2]`); not a trap, just wrong data. `list<string>` works. |
| `list<color>` (list<enum>) | Guest emits `(Color)*(byte*)ptr` which C# parses as multiplication, not a cast (CS0119). |
| empty string `""` | Guest traps in `free()` on a zero-length string. Non-empty strings work. |
| `tuple<...>` returns (`echo-tuple`, `swap-pair`) | Host generator references undefined `result_0` in tuple-return lowering (CS0103). Tuple params alone would marshal, but these functions also return tuples. |
| `option<string>` param | Host generator emits `value.Value` on a `string?` (CS1061). |
| `result<T,E>` (`safe-divide`, `parse-number`, `format-number`) | Guest generator maps result returns to `object` and emits invalid lowering. |
| `flags permission` (echo / has-read) | Guest generator emits `EchoPermission(int)` vs `Permission` partial and `return default`. |
| variant **parameters** (`echo-small-value`, `extract-small-byte`, etc.) | Guest generator references an undefined `v` (no param-lift code). Variant *returns* work. |
| `large-enum` (variant with string payload) returns | Guest post-return uses `(LargeEnum.Case)*(byte*)ptr` (CS0119, same cast-vs-multiply bug). |
| resource `counter` (own/borrow, ctor, methods, static, list<borrow>, option<resource>) | A resource declared in an *exported* interface is generated by the guest as an *imported* resource (DllImport to host); not round-trippable. Host own-resource returns are also known-broken. |
| imports `host-add` / `host-get-name` | Guest generator emits interface-level imports as exports with no DllImport call-out path, so the guest cannot invoke a host import. |

The `commons.wit` still defines `permission` and `large-enum` (unused by any function) so the
type generators still exercise flags/many-case-variant **declaration** lowering.
